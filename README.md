Этот проект - шуточный онлайн-сервис для путешествий, который представляет из себя "маркетплейс" дорогих авиабилетов и плохих туров. Функционал зависит от одного из двух выбранных пользователем типов аккаунта: пользовательский или бизнес (для турагентов и авиакомпаний).

В пользовательском аккаунте есть три вкладки: дорогие авиабилеты, плохие туры и профиль. В авиабилетах и турах есть возможность поиска, просмотра, выбора и покупки позиций (либо авиабилет, либо тур). Авиабилет представляется в виде аккаунта компании, выбора рейса и мест разной ценовой категории (изначально открыты только самые дорогие). Тур представляется в виде аккаунта компании, описания, маршрута в виде списков стран, городов и достопримечательностей, а также фотографий.
Тур или авиабилет можно подарить своему другу (по почте найти его аккаунт). При выборе билета можно подарить другу мистери бокс (рандомные билет в рандомное место).

В профиле отображается имя пользователя, электронная почта, фотография и ачивки: отдельно полученные и не полученные пользователем в зависимости от его действий в сервисе (пропишем их потом в расширенном функционале). Также в профиле отображается количество баллов, заработанных пользователем за те или иные активности в сервисе, за которые можно покупать ачивки (под каждой ачивкой написана ее цена в баллах). Ачивки будут давать доступ к различным функциям сервиса: дополнительные буквы в клавиатуре для поиска, более дешевые места при выборе билетов, отдельные достопримечательности/города/страны, через которые могут проходить туры).

В бизнес-аккаунте будут те же самые три вкладки (профиль, авиабилеты, туры), но в авиабилетах и турах есть две подвкладки: "Мои авиабилеты" и "Все авиабилеты" (или "Мои туры" и "Все туры"). В подвкладках "Все…" находится аналогичный пользовательскому функционал, а в подвкладках "Мои…" – список из туров/авиабилетов, выложенных пользующейся сервисом компанией, а также кнопка "+" для добавления тура/авиабилета. При добавлении авиабилета требуется ввести информацию о рейсе (дата, время и место отправления и прибытия, категории билетов и их цены). При добавлении тура необходимо ввести всю информацию о нем: описание, маршрут (в виде списков стран, городов и достопримечательностей) и также фотографии.

## Запуск

### Бекенд (FastAPI, порт 8000)

```bash
# Локально через SQLite
pip install -r requirements.txt
uvicorn app.main:app --reload

# Через Docker Compose (Postgres)
make run-local
```

Документация API: http://localhost:8000/docs

### Фронтенд (React Native Web, порт 8081)

```bash
cd front
yarn install
yarn web
```

Откройте http://localhost:8081

Фронтенд проксирует запросы к `/auth`, `/tours`, `/users` на `http://localhost:8000` через webpack dev server proxy. Убедитесь, что бекенд запущен перед стартом фронтенда.
